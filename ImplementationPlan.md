## 功能规划：

+ 文化多元化纲要：依据区域、国家的实际情况制定刚性框架规则，由区域自主提案，三方协商(用户方、政府方、鸢尾)并经行政审核通过后该区域系统开始生效。区域没有提案则按默认纲要执行，默认纲要会在以后的更新中附上，所有生效提案也会在相关目录展示。
+ 社交伦理联盟。届时会与同行业的友商与合作伙伴建立起社交伦理联盟，伦理数据联盟内互通。共同帮助现有IT社交行业摆脱滥交、道德低的负面形象，让用户在保持良好社交体验的过程中同时不破坏道德，帮助用户维系道德。使伦理数据对实名用户可兑责、可追溯，部分伦理数据过于混乱的群体会被系统自动标记为滥交用户。伦理数据实际上不存在隐私问题，即便是真实世界的交往也不会密不透风，只要出了些动静交往方的亲朋好友基本都会揭露事件。计算机相对人来说是更靠谱的，只要不出事故不会有人去披露详情。普通用户只能查看到汇总数据，详细数据只有在发生刑事案件时才会调用。
+ 视发展阶段采取全球化、多语言、多风格。
+ 跨区域、国家的对接定制社交，以满足不同人群的不同需求，受区域纲要影响。
---
+ 安全机制。确保信息的现实与线上的数据一致性和真实性。安全报警功能，配备可报警贴身穿戴设备以保障人身安全，要求设备即便在私密空间或进行亲密行为而行动不便时也能迅速发出求救讯号。硬性规定设备必须有方便、可控、用于主动触发警报的物理模块，并必须附上该设备难以手动触发警报的姿势规劝和警示。对于手动触发警报、设备异常脱落、行程点超定量时间未达等异常情形，可设定警报自动触发、联系紧急联系人。用户可以根据自身经济条件购买功能更丰富强大的生命体征监测设备来更好保护自己的安全，比如智能手环、智能手表、定制设备等，当设备监测到心率、血压等体征异常时触发报警，只要数据类似恐慌、死亡等特点就会被判定满足警报条件。有特殊需求的用户能深度定制专属的生命体征警报条件，如果不能独自定制有专业医学知识要求的触发条件，可以付费请专业团队私人定制。
+ 刚性准入条件，受区域纲要影响。
+ 系统层面的指标体系，监管和调控，受区域纲要影响。
+ 健康筛汰机制。筛选掉患有传染病的人群，确保人员交往的健康安全，彻底消灭艾滋病等可防性病。用户可以选择定制生理期数据，进一步设置在生理期内的社交功能调整或禁用，异性伴侣可订阅推送。
+ 契约履责机制。针对纠纷、意外等问题事先协定好责任承诺，事先形成契约锁，事件触发后按协议内容履责。
+ 系统凋零机制。不可否认的是做得再精致的系统都会存在漏洞，只要被人为利用来作恶，任何系统都会失去初衷。保持初衷需要有人不断维系，但保护的人并非总会理所当然的存在。所以当某个地区的破坏和滥用情况已经失控，那么该地区的系统服务会根据指标逐步关闭、清算余款。目的是不妨碍任何一人的自由、不造成负面影响，确保系统存活期间只对受众有利；逻辑是系统与人发生冲突时，优先给人让步，人可以在行为负责的情况下中止系统，但系统不会伤害人。
+ 系统黑名单。对于不懂尊重的群体采取不予准入的封禁措施，且与实名信息唯一对应。

+ 社交匹配。过滤筛选社交目标，达到交往要求后形成关系（普通朋友、恋人、熟友等）。
+ 人际关系管理系统。阶段关系的进退只由用户本身决定，不会采用任何数值来评定关系的亲密程度。系统只对关系作划分和记录。对用户所有交往的关系进行数据治理，达到一定程度的恋人可以被授权了解伴侣的关系状况，可了解交往概况、历史、趋势等。
+ 亲密出行。规划出行计划，定制计划的行程点、付款方式（消耗共同资金账户、AA、女免等），伴侣可参与方案制定，或指定完成后查阅修改等形式，共同决定此次的出行。
+ 行程方案生态。出行方案可在线编辑、分享，私人行程方案库、公共行程方案库。
+ 资金部分。亲密关系建立起以后会开启共同资金账户，双方共同确认资金的分配方式，关系破裂后按资金分配方式归还。资金限制数额在刚建立的关系中较小，限着随关系加深提高。任一方用户的资金流出达到纠纷预警条件（较大数额）时，双方必须需要共同签订关系破裂条款，防止今后大概率会出现的情感纠纷，也防止诈骗案的发生。对支出进行粗略分类，分人情往来、日常开支。人情往来部分存在数额限制；日常开支主要指交往过程中的必要开销，比如餐饮、酒店住宿、火车飞机票等，根据双方拟定的支付方案从共同资金账户中扣款。
+ 情感小结系统。仅当事人双方可见，但一段新关系开始之前双方必须把上一段情感处理妥当，并给出小结和结束时的关系状况，确保无矛盾后才能开启下一段关系。目的是避免引起情感矛盾纠纷，如果违反规则将对账号处置。
+ 个人信息平台。写真，才艺作品，兴趣特长，隐私资料开放对象的权限设置等。
+ 反色情系统。反色情含三块内容，一块是性交易，一块是信息色情，一块是纵欲。1）性交易的特点是每天更换多个伴侣，所以能通过一人一号的实名制、限制更换伴侣的最小周期解决。即便最小周期设置为几天，性交易也将无利可图，无利可图的事很少有人去做，要不结婚也不会那么困难。2）性不意味着色情，性是人之常情，但色情不是。性可以带给人快乐，而色情不会，色情蔓延对人造成的名誉伤害往往会伴随终生。成人社交理应允许性的存在，但不应该允许色情的存在。性的发生应该存在于由男女双方的自由意志而自愿协定的私人隐私场合，绝不应该将不恰当的信息暴露于公共场合。3）通过系统自动化及量化管理来防止纵欲的发生，出于对用户负责的目的，系统会为每个意愿用户定制合适的方案。满足传染病体检和最小伴侣更换周期的硬性条件属于必须，但是否纵欲不属于必须。如果用户自爱应该自觉遵守，生效后系统就会按周期内不可更改的流程执行，这个周期可以自行挑选。

## 现有社交软件的问题
目前绝大多数的社交软件更多地注重考虑人与人之间的线上连接，一个始终没有解决的核心问题在于虚拟与现实的沟壑仍然太大。没有考虑即便人知道了，再到线下社交时会有诸多障碍。而线上再如何甜言蜜语，远不及一个孤独时面对面的温暖拥抱、一双失落时坚定的信任眼神，也不及一件冷时的厚实毛衣、一份饿时的热腾便当。线上的表达不等于线下的顺畅交流，由脱离现实的表达而堆砌的情感到线下很可能会变成尴尬的互视，经济因素、语言不通、虚拟表达无法转化（表情包、斗图）等。所以鸢尾打算从这些痛点着眼，构建起一个易于现实交往的生态。从交往初始就发掘人们现实交往的关注点、兴趣点，并把这些现实愿望通过行程方案来表述，在行程方案生态中可以自由构建心里想要并且切实可行的出行生活，也可以参考别人的优秀计划。包括当事人考虑不到的因素鸢尾会尽力替用户考虑到，比如安全问题、中间语言、场景用到的词汇和知识范围、交通、交往的后续意愿和计划、长期行程的就业推荐、手续证件等。总之，鸢尾会努力让人更容易实现内心对所爱的向往，降低现实因素的障碍。

匹配成功后双方均同意可成为联系人，关系性质为普通朋友，开放通用社交功能，健康状况未核实。<br><br>
在健康状况已核实的情况下，满足彼此阶段交往的要求（交往多久、交往程度、生育愿意、子女抚养等）、双方同意后可将关系性质提为性伴侣（实质如此，名称暂定），开放成人社交功能。一般情况下人员之间不存在法定关系时，性伴侣在同一时间有且只能有一个（协议家庭、夫妻可能会有差异，视地区法律而定），性伴侣关系建立后，与其他人成为性伴侣的通道关闭。关系结束的时间过短时，系统会产生一个性病阻断期，更换性伴侣的频率受区域规则限制（如1个月、2个月）。性伴侣关系无异常结束后，用户功能会重置为无伴侣状况，健康状况会回退至未核实。如果女方怀孕则无法常规地结束关系，无论男方还是女方都无法单方面中止，关系会进入孕养关系并开启生育流程，会实名地对人员数据追踪（不妨碍隐私，用作事故发生时的社会救援），状态面板会显示相关信息，直至责任划分承担无误、流程顺利结束才允许中止关系。孕养关系时可以提升关系。<br><br>
满足协定的交往时间后可以自愿选择成为密友（实质如此，名称暂定），开放亲密社交功能。既是亲密关系又同时是性伴侣关系时，满足彼此的要求后(交往意向等)可成为情侣，开放情侣社交功能。在拥有情侣后密友的数量会根据情侣意愿限制（有要求不满足则无法成为情侣），情侣在同一时间有且只能有一个。如果长时间异地又不愿结束亲密关系，可以授权彼此性伴侣寻觅权限，交往时会有信息显示（当事人可知情，对不相关的人隐蔽），但这不妨碍性伴侣在同一时间有且只能有一个的逻辑和其他规则（健康状况依旧需要核实）。关系可选择只解除情侣或连同性伴侣一并解除，用户功能会重置为相应状况，性伴侣关系结束则健康状况会回退至未核实。

> 两性社交只提供正式的性社交志愿者交友平台和系统对接的满足安全标准的行程点。对行程点的场合性质分类管理，各种性质的场合功能和安全标准有所区分，包括合规的成人场合和生活日常的商家（如餐馆、棋牌等）。普通场合供日常交往，普通成人场合供成年人交往，成人旅宿场合供成年人亲密交往，非强迫性的一同进入成人旅宿场合默认视作发生亲密行为。匹配交往信息只能通过政府承认的信息平台，在系统生成数据记录后按规范处理，接受所有人员自由的辞去身份。无论是线上平台的信息匹配还是线下的实时匹配，都需要满足并达成数据的一致性，线上平台的主要作用是提供交往信息匹配和查阅真实的关键数据，在系统内匹配登记后可至合规的行程点交往。场所分公共与民营，公共场所持非盈利目的，主要作用是供正常交友和在引流至成人旅宿场合前使信息可追溯以保障人身安全和双方的充分自愿，若交流后双方都有进一步交往意向，完整登记必要信息后可至合规的成人旅宿场合。公共场所只建设示范点，以安全和节约为向导，该有的不能省，不该有的不浪费，简洁不简陋，大方得体即可；民营场所在满足安全和隐私标准和规范后就可以对接信息平台，主要职能是旅宿，附加职能由企业自身发挥创造力（如发起活动等服务），效益由市场决定，盈亏自负，总体来说互利互惠，平台为商家带来人流量，商家为平台完善交友生态；与酒店行业合作，诚邀符合安全和隐私规范标准的酒店入驻，不需要全部建造房屋，如此可以节约成本。为了使每个人都能拥有平等的机会，平台不收取费用，不存在高额支出的名目，不存在支付给私人费用的名目，费用均由自身对场所环境的要求高低而产生，其余皆为不作要求的私下的人情关系（资金建议走信息系统，避免财务纠纷）；为了鼓励成为志愿者后在规范场合交往，在培养群众良好行为习惯的初期，双方均是志愿者可减少部分开销，政府可通过减免部分税项补贴给商家。所有进入场合的人员均会被保障隐私安全地记录必要信息，一是减少事故发生，二是便于用作矛盾仲裁的依据。信息登记整合在支付环节，进入场合时NFC刷卡或者扫码，离开时自动结账，产生事故时可以联合旅宿信息作为调查依据，仅供警方调查用，人员在不违反规定的情况下自愿、自由交友，交往不局限于志愿者，非志愿者也可在场合内交往，但必须均成为志愿者才能引流至成人旅宿场合。
